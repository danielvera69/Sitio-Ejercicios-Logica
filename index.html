¿Qué es un Ciclo? El Superpoder de Repetir

Imagina que tienes que realizar una tarea muchas veces, como aplaudir 100 veces. Sería agotador escribir "aplaudir" 100 veces. Un ciclo es la forma en que le decimos a la computadora: "Repite esta acción hasta que yo te diga que pares".

Los 3 Ingredientes de Todo Ciclo

Inicialización

¿Dónde empezamos a contar? Es nuestra línea de partida. (Ej: `contador = 1`)

Condición

La regla que decide si seguimos. Mientras sea cierta, el ciclo continúa. (Ej: `contador <= 100`)

Actualización (Paso)

El paso que damos en cada repetición para acercarnos al final. (Ej: `contador = contador + 1`)

PSeInt vs JavaScript: El ciclo `Para` en PSeInt es como el `for` de JavaScript, agrupa los 3 ingredientes. El `Mientras` de PSeInt es como el `while` de JavaScript, donde los ingredientes están más separados pero cumplen la misma función.

La Estrategia Maestra: Piensa Antes de Programar

El error más grande es intentar escribir código sin un plan. Sigue estos 4 pasos siempre, y verás cómo los problemas se vuelven más sencillos.

  1. 1 Resuelve a Mano (Simulación): Olvida la computadora. Escribe el resultado que esperas como si lo hicieras en un cuaderno.
  2. 2 Encuentra el Patrón (Detective): Mira tu solución manual. ¿Qué acción se repite? ¿Qué valores cambian en cada paso? Esta es la parte más importante.
  3. 3 Asigna Variables (Etiquetas): Dale un nombre a cada "pieza" importante de tu patrón: el contador, el límite, la suma, etc.
  4. 4 Define las Reglas del Ciclo (El Plan): Usando tus variables, define claramente los 3 ingredientes: ¿dónde Inicia?, ¿cuál es la Condición para seguir?, ¿cómo se Actualiza en cada paso?

Análisis con la Estrategia Maestra

1. Simulación Manual (entre 1 y 6):
• Reviso 1: no par.
• Reviso 2: par. Guardo 2 en "caja de suma". Suma=2.
• Reviso 3: no par.
• Reviso 4: par. Añado 4. Suma=2+4=6.

2. Patrón Detectado: Mismo ciclo y filtro que el ejercicio 8, pero la acción dentro del filtro cambia. En lugar de "escribir", la acción es "sumar a un total". A esta variable que guarda totales la llamamos acumulador.

3. Variables:
• num1, num2, c.
• suma: Nuestro acumulador. ¡Debe empezar en 0!

4. Reglas del Ciclo:
• Antes del ciclo: Inicializar suma = 0.
• Ciclo: Contador c va desde num1 hasta num2.
• Dentro del Ciclo: Si c es par, entonces actualizamos el acumulador: suma = suma + c.
• Después del ciclo: Mostrar el valor final de suma.

Código de Ejemplo

Proceso SumaPares
    Definir num1, num2, c, suma Como Entero
    num1 <- 1; num2 <- 10; suma <- 0
    Para c <- num1 Hasta num2 Hacer
        Si c MOD 2 = 0 Entonces; suma <- suma + c; FinSi
    FinPara
    Escribir "La suma es: ", suma
FinProceso
function sumaParesFor(num1, num2) {
  let inicio = Math.min(num1, num2), fin = Math.max(num1, num2);
  let suma = 0;
  for (let c = inicio; c <= fin; c++) {
    if (c % 2 === 0) { suma = suma + c; }
  }
  console.log(`La suma es: ${suma}`);
}
sumaParesFor(1, 10);
function sumaParesWhile(num1, num2) {
  let c = Math.min(num1, num2), fin = Math.max(num1, num2);
  let suma = 0;
  while (c <= fin) {
    if (c % 2 === 0) { suma += c; } // += es atajo de suma = suma + c
    c++;
  }
  console.log(`La suma es: ${suma}`);
}
sumaParesWhile(1, 10); // 2+4+6+8+10=30

Análisis con la Estrategia Maestra

Definición: Un número es perfecto si la suma de sus divisores PROPIOS (todos menos él mismo) es igual al número.

1. Simulación Manual (para 6): Divisores propios: 1, 2, 3. Suma = 1+2+3=6. ¿Es la suma (6) igual al número original (6)? Sí. Es perfecto.

2. Patrón Detectado: Es casi idéntico al ejercicio 11, pero con dos cambios: (1) el ciclo para buscar divisores debe parar uno antes (hasta `num - 1`), y (2) al final, debemos comparar la suma con el número original.

3. Variables: Las mismas que el ejercicio 11.

4. Reglas del Ciclo:
• Antes: suma = 0.
• Ciclo: Contador i va de 1 hasta num - 1.
• Dentro: Si i es divisor, suma = suma + i.
• Después: Comparar: Si (suma = num), es perfecto. Si no, no lo es.

Código de Ejemplo

Proceso NumeroPerfecto
    Definir num, i, suma Como Entero;
    num <- 28; suma <- 0
    Para i <- 1 Hasta num - 1 Hacer
        Si num MOD i = 0 Entonces; suma <- suma + i; FinSi
    FinPara
    Si suma = num Entonces Escribir "Perfecto" Sino Escribir "No es perfecto"; FinSi
FinProceso
function esPerfectoFor(num) {
  let suma = 0;
  for (let i = 1; i < num; i++) {
    if (num % i === 0) { suma += i; }
  }
  if (suma === num && num !== 0) {
    console.log(`${num} es perfecto.`);
  } else {
    console.log(`${num} NO es perfecto.`);
  }
}
esPerfectoFor(28);
function esPerfectoWhile(num) {
  let suma = 0, i = 1;
  while (i < num) {
    if (num % i === 0) { suma += i; }
    i++;
  }
  if (suma === num && num !== 0) {
    console.log(`${num} es perfecto.`);
  } else {
    console.log(`${num} NO es perfecto.`);
  }
}

Análisis con la Estrategia Maestra

Definición: Un número primo solo tiene 2 divisores: 1 y él mismo.

1. Simulación Manual (para 7): Divisores: 1, 7. Total: 2. -> Es primo.
Simulación (para 9): Divisores: 1, 3, 9. Total: 3. -> No es primo.

2. Patrón Detectado: La estrategia no es sumar, sino contar cuántos divisores tiene un número. Usaremos una variable para llevar esta cuenta.

3. Variables:
• num, i.
• cantDivisores: Un contador de eventos. Empieza en 0.

4. Reglas del Ciclo:
• Antes: cantDivisores = 0.
• Ciclo: Contador i desde 1 hasta num.
• Dentro: Si i es divisor, incrementamos cantDivisores++.
• Después: Si cantDivisores es exactamente 2, el número es primo.

Código de Ejemplo

Proceso NumeroPrimo
    Definir num, i, cont Como Entero;
    num <- 7; cont <- 0
    Para i <- 1 Hasta num Hacer
        Si num MOD i = 0 Entonces; cont <- cont + 1; FinSi
    FinPara
    Si cont = 2 Entonces Escribir "Primo" Sino Escribir "No es primo"; FinSi
FinProceso
function esPrimoFor(num) {
  if (num <= 1) { console.log("No es primo"); return; }
  let cantDivisores = 0;
  for (let i = 1; i <= num; i++) {
    if (num % i === 0) { cantDivisores++; }
  }
  if (cantDivisores === 2) { console.log("Es primo"); } 
  else { console.log("No es primo"); }
}
esPrimoFor(7);
function esPrimoWhile(num) {
  if (num <= 1) { console.log("No es primo"); return; }
  let cantDivisores = 0, i = 1;
  while (i <= num) {
    if (num % i === 0) { cantDivisores++; }
    i++;
  }
  if (cantDivisores === 2) { console.log("Es primo"); } 
  else { console.log("No es primo"); }
}
esPrimoWhile(29); esPrimoWhile(30);

• 1 >= 3? No. Parar.
• Resultado: 3 (cociente) y sobra 1 (residuo).

2. Patrón Detectado: Mientras el dividendo sea mayor o igual que el divisor, restamos y contamos. El ciclo `while` es ideal aquí, porque no sabemos de antemano cuántas restas haremos.

3. Variables:
• dividendo, divisor.
• cociente: Contador de restas. Empieza en 0.

4. Reglas del Ciclo:
• Antes: cociente = 0.
• Ciclo (Condición): Mientras dividendo >= divisor.
• Dentro: dividendo = dividendo - divisor y cociente++.
• Después: Mostrar `cociente` y el `dividendo` sobrante (residuo).

Código de Ejemplo

Proceso DividirRestando
    Definir dividendo, divisor, cociente Como Entero
    dividendo <- 10; divisor <- 3; cociente <- 0
    Mientras dividendo >= divisor Hacer
        dividendo <- dividendo - divisor
        cociente <- cociente + 1
    FinMientras
    Escribir "Cociente: ", cociente, " Residuo: ", dividendo
FinProceso
// Forzar un ciclo 'for' es muy poco natural para este problema.
// 'while' es la herramienta correcta.
function dividirRestandoFor(dividendo, divisor) {
  let cociente = 0;
  for ( ; dividendo >= divisor; cociente++) {
    dividendo -= divisor;
  }
  console.log(`Cociente: ${cociente}, Residuo: ${dividendo}`);
}
dividirRestandoFor(10, 3);
function dividirRestandoWhile(dividendo, divisor) {
  if (divisor === 0) { console.log("Error: división por cero."); return; }
  let cociente = 0;
  while (dividendo >= divisor) {
    dividendo -= divisor;
    cociente++;
  }
  console.log(`Cociente: ${cociente}, Residuo: ${dividendo}`);
}
dividirRestandoWhile(22, 5);
